# ウクレレ コード譜ジェネレーター

`コード表.xlsx` を編集して push するだけで、ウクレレのコードダイアグラム画像が自動生成され、GitHub Pages上の一覧サイトが更新されます。

## 仕組み

```
コード表.xlsx を push
        │
        ▼
GitHub Actions が起動 (.github/workflows/generate.yml)
        │
        ├─ scripts/generate_chords.py を実行
        │     ・Excelのセル値(コード名・フレット番号)を読み取り
        │     ・xlsx内部の図形(○=押弦、太い縦長=セーハ)を解析
        │     ・コードごとに docs/chords/<コード名>.png を生成
        │     ・docs/manifest.json (コード一覧データ) を生成
        │
        ├─ 生成した画像をリポジトリに自動コミット
        │
        └─ docs/ を GitHub Pages に自動デプロイ
                │
                ▼
        公開サイトが最新の状態に更新される
```

## 使い方

1. **`コード表.xlsx` を編集する**
   既存のシート(C, C#・D♭, D... など)の中で、コードを追加・修正する。
   - コード名のセルの下に「1, 2, 3, 4, 5」というフレット番号の行があるブロックを1セットとして認識します。
   - 押弦位置は、セルの上に○の図形(楕円)を重ねて表現します。
   - セーハ(複数弦を1フレットで押さえる)は、縦長の角丸長方形(フローチャートの「端子」図形)を90度回転させて配置します。
   - 既存のコードを1つコピーして、名前と○の位置を書き換えるのが最も簡単です。

2. **GitHubにpushする**
   ```
   git add コード表.xlsx
   git commit -m "コードを追加"
   git push
   ```

3. **数十秒〜1分待つ**
   GitHub Actionsが自動で画像を生成し、サイトを更新します。
   進行状況は リポジトリの「Actions」タブ から確認できます。

4. **サイトを確認する**
   `https://<ユーザー名>.github.io/<リポジトリ名>/` で一覧を確認できます。

## 初回セットアップ(リポジトリ管理者向け)

1. このリポジトリの内容をGitHubの新規リポジトリにpushする。
2. リポジトリの **Settings → Pages** を開き、
   - Source: 「GitHub Actions」 を選択する。
3. **Settings → Actions → General → Workflow permissions** で
   - 「Read and write permissions」 を選択する。
   (画像を自動コミットするために必要です)
4. `main` ブランチに何かpushすると、初回のビルド・デプロイが走ります。

## ローカルで画像生成を試す場合

```bash
pip install openpyxl pillow
python scripts/generate_chords.py
```

`docs/chords/` に画像、`docs/manifest.json` に一覧データが出力されます。

## ファイル構成

```
コード表.xlsx              ← 編集対象のExcelファイル
scripts/
  generate_chords.py       ← 画像生成スクリプト
docs/
  index.html               ← 一覧表示サイト(GitHub Pagesで公開される)
  manifest.json            ← コード一覧データ(自動生成)
  chords/                  ← 生成されたコード画像(自動生成)
.github/workflows/
  generate.yml             ← 自動化の設定(GitHub Actions)
```

## コードブロックの認識ルールについて

`generate_chords.py` は、シート内をスキャンして「連続する整数5つ(例: 1,2,3,4,5 や
10,11,12,13,14)が並ぶ行」をフレット番号行として検出し、その直前にあるNUM_STRINGS行分
(ウクレレなので4行)をコードのグリッド本体として扱います。さらにその数行上にある
文字列セルをコード名として扱います。

新しいルート音のシートを追加する場合も、既存シートと同じレイアウト(コード名 → 空白 →
4行のグリッド → フレット番号の行)を保てば、自動で認識されます。

### ハイポジションのコードについて

フレット番号が「1,2,3,4,5」ではなく「10,11,12,13,14」のように1以外から始まる場合、
そのコードはハイポジション(高いフレット)のフォームとして扱われます。画像では:
- ナット(0フレット)の太線は表示されない
- グリッド左に、開始フレット番号(上の例では「10」)が小さく表示される

という、一般的なギター/ウクレレ教材と同じ表記になります。

## 重複コードの扱いについて

同じシート内に同名のコードが複数見つかった場合、スクリプトは押弦位置と開始フレット番号まで
比較します。

- **押弦位置・開始フレットが完全に同じ場合** → コピーミスとみなし、自動的に1つだけ残します。
- **押弦位置か開始フレットが異なる場合**(同じコード名で別の運指・別のポジション) →
  両方を別ファイルとして残し、実行時のログに
  `[情報] 同名だが運指が異なるコードが見つかった...` と表示します。
  この場合はExcel側で本当に2種類用意したいのか、入力ミスなのかをご確認ください。

実行結果は標準出力(GitHub Actionsの場合はActionsのログ)に出力されます。
